# DEADGL v1.2 Geometry Pipe

This branch is reserved for the next real renderer cut.

## Contract

DEADGL remains a CPU framebuffer command machine:

```text
command -> memory -> image -> hash
```

v1.2 extends the command stream with a real geometry path. It must not become a game engine wrapper or OBJ tutorial clone.

## Required changes

- Bump version to `1.2.0` only when code and tests pass.
- Add camera state to scene execution.
- Add projected triangle commands:

```text
camera EX EY EZ TX TY TZ FOV
face X0 Y0 Z0 X1 Y1 Z1 X2 Y2 Z2 COLOR
wireface X0 Y0 Z0 X1 Y1 Z1 X2 Y2 Z2 COLOR
```

- Keep `ztri` as the raw screen-space depth-tested triangle path.
- Add a cube demo that is generated by CPU projection, not pre-rendered art.
- Add `examples/cube.dgl` using `camera`, `face`, and `wireface`.
- Update scene format docs.
- Update tests so unknown geometry commands fail before implementation and pass after implementation.
- Update local release script to emit `dist/deadgl-1.2.0`.
- Add v1.2 release notes.

## Non-goals

- No OBJ loader yet.
- No texture loader yet.
- No GPU backend.
- No scene graph.
- No hidden engine layer.

## Done means

```sh
make clean test
make clean sanitize
make clean
make
build/deadgl demo cube -o build/cube.ppm
build/deadgl run examples/cube.dgl -o build/cube_scene.ppm
build/deadgl prove examples/cube.dgl -o build/cube_scene.ppm -p build/cube_scene.proof
```

The proof hash must be deterministic across repeated runs on the same source tree.
